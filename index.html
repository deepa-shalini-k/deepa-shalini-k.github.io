Product Vision and Strategy ‚åÑ

Product Vision

ChaRtBot is a visual analytics assistant that helps individuals explore and understand data through fast, interactive charts ‚Äî using natural language instead of manual coding or rigid dashboards.

The vision is to make data exploration visual-first, reliable, and transparent, enabling users to move from raw data to insight in seconds while keeping humans firmly in the loop through dataset inspection, generated code visibility, and exportable outputs.

Unlike text-based AI tools, ChaRtBot responds with interactive visualizations rather than paragraphs. Unlike most analytics tools, it is web-based, lightweight, and requires no setup, installation, or template configuration.

Product Strategy

ChaRtBot is intentionally designed around four core principles:

1. Visual-first by default

The primary output is always a chart, not text. Visuals enable faster pattern recognition and decision-making.

2. Speed as a feature

Charts render in seconds, supporting rapid exploration, prompt refinement, and iteration.

3. Transparency and trust

Users can inspect the underlying dataset, view the generated code, and export interactive outputs ‚Äî ensuring clarity and human oversight at every step.

4. Individual-first adoption

ChaRtBot is built for personal use cases such as research, learning, exploratory analysis, and lightweight project work, without the complexity or overhead of enterprise BI tools.

Roadmap ‚åÑ

Product Roadmap

ChaRtBot is being developed in phases, with each stage focused on improving reliability, clarity, and analytical depth.

Phase 0: Current MVP

Goal: Deliver instant, shareable visual insights from user-provided data

Capabilities

  • CSV upload with interactive data table (sorting, filtering, scanning)
  • Natural-language prompts to generate interactive charts
  • Support for complex subplotting and multi-chart layouts
  • Visibility into the generated Python code
  • Interactive HTML export for sharing and reuse
  • Chart image export
  • Web-based, link-only access (no installation required)
  • Chart rendering within seconds

User Value

  • Faster visual exploration compared to text-based AI tools and desktop software
  • Shareable, interactive outputs suitable for projects and presentations
  • Transparency through dataset inspection and code visibility
Phase 1: Explainability & Reliability

Goal: Expand accessibility to non-technical users while increasing trust and robustness

Capabilities

  • Plain-language explanations of generated code
  • Visual explanations of data aggregations and transformations
  • Clear communication of assumptions made by the system
  • Verification checks before execution (e.g. schema and column validation)
  • Multi-pass code correction to recover from errors
  • Fallback chart strategies when complex visualizations fail
  • User confidence checks and satisfaction feedback

Why this matters

  • Reduces failed chart generations
  • Makes analysis understandable to non-Python and non-technical users
  • Maintains high transparency while improving reliability
Phase 2: Conversational Refinement & Intent Clarification

Goal: Handle ambiguity before it results in incorrect or failed outputs

Capabilities

  • Detection of ambiguous or incomplete prompts
  • Targeted follow-up questions to clarify intent
  • Explicit declaration of assumptions when clarification is skipped
  • Suggestions for more suitable chart structures

User Value

  • Fewer incorrect charts
  • More collaborative, analyst-like interaction
  • Reduced need for trial-and-error prompting
Phase 3: Advanced EDA & Statistical Analysis

Goal: Move beyond chart generation into deeper analytical reasoning

Capabilities

  • Exploratory data analysis (distributions, correlations, outliers)
  • Statistical analysis with visual explanations
  • Automatic EDA flows for large or complex datasets
  • Advanced techniques such as PCA, clustering, and dimensionality reduction
  • Chart recommendations based on data characteristics

User Value

  • Enables complex mathematical and statistical analysis through visuals
  • Suitable for research-grade and large-dataset exploration
  • Positions ChaRtBot as a visual analysis companion rather than a charting tool
Phase 4: Machine Learning‚ÄìAugmented Analysis

Goal: Enable machine learning‚Äìdriven reasoning for large datasets, grounded in thorough exploratory analysis and communicated through visualizations.

Capabilities

  • Scalable exploratory data analysis for large datasets
  • Guided transition from EDA to modelling
  • Automatic selection of appropriate ML approaches based on data characteristics
  • Model training for prediction and classification tasks
  • Accuracy evaluation using visual diagnostics (e.g. error distributions, confusion matrices)
  • Clear reporting of accuracy, model performance, limitations, and assumptions

User Value

  • Enables predictive and inferential analysis without manual model selection
  • Ensures ML decisions are explainable and justified, not opaque
  • Helps users understand why a model was chosen and how well it performs
Phase 5: Real-Time & External Data Integration

Goal: Reduce dependency on manually uploaded datasets

Capabilities

  • Ability to pull real-time or public data from the web
  • Clear citation and source visibility
  • Data augmentation by combining uploaded datasets with external data
  • Freshness and provenance checks

User Value

  • Faster insights without manual data collection
  • Support for time-sensitive and contextual analyses
  • Broader analytical possibilities beyond static files
Phase 6: Publication-Grade Outputs & Sharing

Goal: Make ChaRtBot the final step before communication and publication

Capabilities

  • Publication-ready chart styling and themes
  • High-resolution exports for papers, posters, and presentations
  • Versioned chart history
  • Shareable links with embedded interactivity

Strategic Impact

  • Completes the workflow from data ‚Üí analysis ‚Üí communication
  • Appeals to researchers, analysts, and professionals who need polished outputs
KPIs and Success Metrics ‚åÑ

ChaRtBot's success is measured across adoption, reliability, and user trust, with additional metrics tracking the uptake and impact of advanced analytical capabilities as the product evolves.

1. Adoption & Engagement

Why: To understand whether users are finding value in ChaRtBot and returning to it as a regular part of their data exploration workflow.

Metrics

  • Weekly Active Users (WAU)
  • Average number of charts generated per session
  • Percentage of sessions with multiple or conversational follow-up prompts (Phase 2+)
  • Usage rate of multi-chart and subplot layouts
2. Performance & Reliability

Why: Speed and reliability are core differentiators. Charts must render quickly and consistently for users to trust and continue using the product.

Metrics

  • Median time to first chart
  • Chart render success rate
  • Error recovery rate after failed executions (Phase 1+)
  • Drop-off rate during chart generation
3. Trust & Transparency

Why: ChaRtBot prioritises human-in-the-loop analysis. These metrics assess whether users are engaging with explanations, verifying outputs, and confidently reusing results.

Metrics

  • Percentage of users viewing generated code
  • Percentage of users viewing non-technical code explanations (Phase 1+)
  • Frequency of HTML and image exports
  • User confidence and satisfaction feedback after chart generation (Phase 1+)
4. Analytical Depth & Advanced Feature Adoption

Why: To measure the product's evolution from basic chart generation to deeper visual reasoning and advanced analysis.

Metrics

  • Percentage of sessions using EDA and statistical features (Phase 3+)
  • Percentage of datasets analysed using machine learning features (Phase 4+)
  • Frequency of model comparison or re-training based on feedback (Phase 4+)
  • User-reported confidence in ML-driven insights (Phase 4+)
5. Retention & Power-User Growth

Why: To understand whether advanced capabilities meaningfully increase long-term value and user retention.

Metrics

  • Retention rate of advanced-feature users vs basic users (Phase 3+)
  • Longitudinal increase in feature depth per returning user
